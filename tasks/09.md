🧪 Testes de Desempenho na API
📌 Objetivo

Este documento descreve como aplicar testes de desempenho em uma API, incluindo:

Teste de Performance

Teste de Carga (Load Test)

Teste de Estresse (Stress Test)

Teste de Banco de Dados

O objetivo é garantir que o sistema não apenas funcione corretamente, mas também seja rápido, estável e escalável.

📁 Estrutura de Pastas

Os testes de desempenho devem ser organizados separadamente dos demais testes:

tests/
  performance/
  load/
  stress/
  database/

Essa separação é importante porque esses testes são mais pesados e não devem rodar com frequência durante o desenvolvimento.

⚡ Teste de Performance
📌 O que é

Mede o tempo de resposta de uma operação individual da API.

🎯 Objetivo

Avaliar tempo de execução de use cases

Medir tempo de resposta do banco

Identificar operações lentas

🧠 Como aplicar

Executar uma função específica (ex: CreateWorkoutPlan)

Medir tempo de execução

⚠️ Importante

Não simula múltiplos usuários

Foco é desempenho individual

🚀 Teste de Carga (Load Test)
📌 O que é

Simula múltiplas requisições simultâneas.

🎯 Objetivo

Verificar comportamento com vários usuários

Identificar gargalos

Avaliar estabilidade

🧠 Como aplicar

Executar várias chamadas simultâneas

Usar Promise.all

Medir tempo total e erros

💥 Exemplo conceitual
100 requisições ao mesmo tempo
🔥 Teste de Estresse (Stress Test)
📌 O que é

Leva o sistema ao limite.

🎯 Objetivo

Descobrir até onde o sistema aguenta

Identificar ponto de falha

Testar estabilidade extrema

🧠 Como aplicar

Aumentar progressivamente a carga

Monitorar erros e lentidão

Identificar quando o sistema quebra

🗄️ Teste de Banco de Dados
📌 O que é

Valida desempenho e consistência do banco.

🎯 Objetivo

Medir tempo de queries

Garantir integridade dos dados

Validar relações entre entidades

🧠 Como aplicar

Executar operações reais no banco

Validar dados retornados

Avaliar eficiência das queries

⚠️ Boas Práticas

Não rodar esses testes junto com testes unitários

Executar separadamente (scripts específicos)

Usar dados controlados

Evitar impacto em banco de produção

Medir tempo com ferramentas apropriadas

🚫 O que NÃO fazer

Não misturar com testes unitários

Não rodar a cada commit

Não usar banco de produção

Não exagerar na quantidade de dados

✅ O que FAZER

Criar pastas separadas

Rodar testes sob demanda

Medir tempo de execução

Simular cenários reais

Monitorar erros

🎯 Resultado Esperado

Com a aplicação desses testes, o sistema será capaz de:

Responder rapidamente

Suportar múltiplos usuários

Manter estabilidade sob carga

Evitar falhas em produção

🧠 Conclusão

Os testes de desempenho são essenciais para garantir a qualidade da aplicação em ambientes reais. Eles complementam os testes funcionais, permitindo avaliar não apenas se o sistema funciona, mas também como ele se comporta sob diferentes condições de uso.
